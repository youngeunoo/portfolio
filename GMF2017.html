    GMF 2017

    GMF 2017의 프로모션 페이지를 기획하고 반응형 웹으로 구현하여 사용자가 다양한 기기로 행사 정보를 확인할 수 있도록 한다.

    개인 프로젝트 개요

    • 작업 기간 : 24. 01. 17 - 24. 01. 21
    • 기여도 : 100%

    목표

    • 새로운 형태의 페이지를 구현하며 레이아웃 구성의 다양성을 높임
    • 다양한 메뉴의 형태를 구현함
    • 반응형 웹 구현을 통해 반응형 웹 작업에 대한 이해를 높이고 다양한 크기의 브라우저에 대응할 수 있도록 함.

    작업 순서

    • 사전 기획
      • 리디자인할 대상 선정
      • 페이지 분석
    • 디자인 작업
      • 메인 색상과 이용 폰트 선정
      • 필요한 이미지와 아이콘 수집
      • 디자인 키워드 선정
      • 디자인 완성
    • 개발 작업
      • 페이지 레이아웃 작업
      • 내용 구성 후 작성
      • header 영역 GSAP 작업
      • 사이드 메뉴의 sticky 작업

    사용 기술 / 도구

    • HTML5
    • CSS3
    • JaveScript
    • GSAP
    • Figma

    디자인 가이드

    • 키워드
      • #활기참
      • #생동감
      • #우주
    • 색상

      • 메인 색상

      • → 우주의 느낌을 주는 색상과 GMF의 대표적인 이미지 색상을 사용하여 컨셉을 전달
    • 폰트

    퍼블리싱

    • 사이드 메뉴 영역에 position:sticky를 적용하여 스크롤시에도 해당하는 영역을 따라올 수 있게 하였다.
    • timetable 영역의 모바일 버전에서는 스테이지별로 선택하여 볼 수 있도록 해 보기 쉽도록 하였다.
    • GSAP ScrollTrigger를 사용하여 상단바의 형태가 위치에 따라 바뀔 수 있도록 하였다.

    문제 및 해결

    • 타임테이블에 첨부된 이미지가 모바일에서 확인하기엔 부적절하여 어떤 식으로 제공해야 할지 고민했다.

      → 각각의 스테이지를 선택할 수 있는 select box를 만들어 원하는 스테이지를 선택하면 해당 스테이지의 타임테이블을 확인할 수 있도록 했다.

    • 태블릿, 모바일 버전에서 메뉴 바가 내려올 때, 메뉴 외 배경은 블러 처리를 하고 싶어 backdrop-filter:blur를 적용하였으나 적용되지 않았다.

      → 이유를 찾아보니 배경 요소가 필요하다고 해서 div로 가상 요소를 추가한 후에 브라우저를 전부 채울 수 있게 크기를 설정하고 적용하니 원하던 효과가 적용되었다.

    • 사이드 메뉴에서 제목을 누르면 각 영역으로 이동이 될 수 있도록 하였는데, 마지막 영역에서 맨 위 버튼을 누르면 원하는 영역으로 이동하지 않았다.

      → 투명하게 설정된 header의 로고와 겹쳐서 해당 버튼이 제대로 클릭되지 않고 있는 게 원인이었고, header의 height 값을 조절하여 해당 문제를 해결하였다.

    후기

    • 아쉬운 부분
      • 애니메이션 부족: festival map 영역의 이미지가 너무 단조롭게 들어가 있음. 직접 이미지까지 구성했다면 애니메이션 효과를 넣어 더 다채롭게 구성할 수 있었을 것 같음.
      • select box 디자인: select box를 디자인 하는 데 한계가 있어 완벽하게 디자인하지 못하였음.
      • 시각적 요소 부족: 시선을 한눈에 사로잡는 요소들이 부족한 것 같음.
    • 잘한 부분
      • 자유로운 반응형 구현: 어떤 사이즈에도 어색하지 않은 디테일한 반응협 웹을 구현했다는 점이 매우 만족스러움.
      • 다양한 메뉴 형태 구성: 다양한 형태의 메뉴들을 페이지에 잘 어우러지게 구현함.
      • 직관적인 정보 전달: 사용자가 행사에 대한 정보를 잘 알 수 있도록 깔끔하고 보기 쉽게 구성함.
    • 배운 점 및 주관적인 의견
      • 기존 작업과는 다른 새로운 형태의 프로젝트를 진행하며 다양한 레이아웃의 경험을 할 수 있어 좋았다.
      • 반응형 웹 작업을 완벽하게 하는 경험이 처음이라 긴장했는데, 데스크탑, 태블릿, 모바일의 큰 틀을 잡고, 디테일한 부분을 잡아나가는 법을 금방 터득하여 뿌듯했다.
      • 정보 전달을 원활히 하기 위해 다양한 형태를 구현해 보는 과정에서 다양한 아이디어를 생각해 볼 수 있었다.
      • 기존 이미지를 활용하여 페이지를 구현하다보니 원하는만큼의 디자인을 끌어내지 못한 경우가 있어 아쉬웠다. 이 점은 계속해서 공부하여 채워나가고 싶다.
