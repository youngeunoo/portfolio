    북구 스토리텔링 협의회

    팀을 구성하여 모든 팀원이 함께 "부산 스토리텔링 협의회"의 페이지를 구현하고, 적극적인 의사소통과 문제해결을 통해 웹 개발에 필요한 기술들을 향상시킨다.

    프로젝트 개요

    • 작업기간 : 23. 10. 26 - 23. 11. 10
    • 기여도 : 50%

    목표

    • 다양한 이미지를 활용하여 디자인을 구현할 수 있다.
    • 팀원들 간의 활발한 소통으로 원활한 의견 교환이 이루어 질 수 있게 한다.
    • 미디어 쿼리를 통한 반응형 웹 작업에 익숙해 질 수 있도록 한다.

    작업 순서

    • 레퍼런스 페이지 조사 / 선정
    • 업무 분담
    • pc버전, 모바일 버전 홈페이지 마크업 작업
    • 홈페이지 분석(요구사항, 필요기능 등)
    • 홈페이지 구현에 필요한 소스들과 assets 폴더 정리
    • 퍼블리싱 가이드 작성
    • 각자 맡은 부분의 페이지 구현 작업
    • 멀티 페이지 구현
    • 발표 준비 / 프로젝트 완료 보고서 작성

    사용 기술

    • HTML5
    • CSS3
    • JaveScript
    • GitHub
    • Figma

    퍼블리싱 가이드

    • 해상도 기준 : PC = 1200px, 모바일 우선 작업
    • 문서에서 각 영역은 각주와 한 줄 띄어쓰기를 이용해 정리
    • 페이지 내 각 영역은 역할에 따라 head, section, footer 태그를 사용

    문제 및 해결

    • swiper 사용 시 각 슬라이드가 움직일 때마다 위치가 달라져 지그재그로 움직이는 효과를 주고 싶었지만 방법을 알지 못했다.

      → 기본 슬라이드 요소에 margin-top의 값을 주고 prev와 next 슬라이드에 margin-top의 값을 0으로 주어서 해결하였다. 코드보기 →

    • 서서히 요소들이 올라오는 이벤트를 구현해야 하는데 어떤 코드를 적용해야 하는지 몰랐다.

      → 검색 끝에 IntersectionObserver 코드를 찾아 요소가 보여질 때 이벤트가 실행되도록 구현하였다.

    후기

    • 아쉬운 부분
      • 의사소통의 부재: 의사소통이 원활하고 자유롭게 이루어지지 못함
      • 약속 시간 미준수: 작업들이 약속된 시간 안에 모두 이루어지지 않았음
      • 발표 준비 미흡: 발표가 제대로 준비되지 않아 듣는 사람이 이해하기 힘들었음
    • 잘한 부분
      • 상호보완적인 협업: 서로 부족한 부분을 채워주고 도우며 작업을 완성해냄
      • 만족스러운 완성도: 본래 페이지와 최대한 비슷하게 구현해냄
    • 배운 점 및 주관적인 의견
      • 팀원 간의 의사소통과 의견 교환이 얼마나 중요한지 깨닫게 되는 과정이었다.
      • 적절한 업무 분담과 프로젝트 진행에 대한 일정 관리가 일찍 정해졌으면 더 좋았겠다는 아쉬움이 남는다.
      • 하나의 프로젝트를 끝까지 끌고 가려면 체력도 매우 중요하다는 걸 깨달았다.
      • 이번 프로젝트에서 느끼고 배운 점을 바탕으로 다음 프로젝트나 작업이 있다면 더욱 많은 시간을 할애하고 노력해서 이번보다 훨씬 더 발전된 개발 기술과 의사소통 기술을 바탕으로 프로젝트를 완성하고 싶다.
