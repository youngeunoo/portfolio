    싱싱고

    냉장고 속 식료품의 소비기한을 한눈에 보고 관리할 수 있는 "싱싱고" 애플리케이션을 구상, 디자인하고 구현한다.

    프로젝트 개요

    • 작업기간 : 23. 12. 01 - 23. 12. 22
    • 기여도 : 100%

    목표

    • 애플리케이션 구현에 필요한 리액트 기술을 학습하고 활용하며 실력을 증진시킬 수 있도록 함
    • 정해진 기간 안에 완성도 있는 애플리케이션을 구현하여 완성의 경험과 자신감을 기름
    • 팀원들 간의 원활하고 적극적인 의사소통을 통해 프로젝트를 더 나은 방향으로 나아갈 수 있게 함

    작업 순서

    • 사전 기획
      • 개발할 애플리케이션의 기능 선정
      • 사용자 시나리오 작성
      • 유스케이스&워크플로우 작성
      • 디렉토리 구조 작성
      • 코딩 및 디자인 가이드 작성
    • 디자인 작업
      • UI 프로토타입 작업
      • 메인 색상과 포인트 색상, 사용될 폰트 선정
      • 애플리케이션에 삽입될 일러스트, 아이콘 작업
      • 전체적인 분위기를 통합하여 UI 디자인 완성 후 스타일 작업
    • 개발 작업
      • 디렉토리 설정
      • 페이지 레이아웃 작업
      • 각 페이지에 필요한 기능 구현

    문제 및 해결

    • 식재료 목록에서 새로고침을 하면 등록한 요소들이 사라지는 문제가 있었다.

      → localStorage를 이용하여 브라우저에 데이터를 저장 후 불러오는 방식으로 유지하였다.

    • 이벤트를 적용하는 달력의 구현을 어떤 식으로 진행해야 할지 갈피를 잡지 못하였다.

      → fullcalendar 라이브러리를 발견한 후 달력 기능과 날짜별로 이벤트를 띄우는 기능을 활용하여 원하는 기능을 구현하였다. 

      import dayGridPlugin from "@fullcalendar/daygrid";
import interactionPlugin from "@fullcalendar/interaction";
import Fullcalendar from "@fullcalendar/react";
import styles from "./Calendar.module.css";
import "./calendar.css";

    // FullCalendar에서 사용할 이벤트 배열 생성
    const eventArray = foodList.map((item) => {
      const endDay = new Date(item.expirationDate);
      const today = new Date();
      const timeDiff = endDay.getTime() - today.getTime();
      const daysDiff = Math.ceil(timeDiff / (100 * 3600 * 24));

      return {
        category: item.category,
        title: item.foodName,
        start: item.purchaseDate,
        quantity: item.quantity,
        endDay:item.expirationDate,
        dDay: daysDiff >= 0 ? `D-${daysDiff}` : D+${Math.abs(daysDiff)}`,
      };
    });

    setEvents(eventArray);

    • 식료품 선택 후 삭제 버튼을 누를 때 랜덤의 식료품이 삭제되는 문제가 있었다.

      → uuid를 이용하여 식료품마다 고유의 id값을 가지게 한 후 해당 식료품의 id를 찾아 삭제할 수 있도록 삭제 함수를 수정하였다.

    후기

    • 아쉬운 부분
      • 완벽한 구현 실패: 처음에 기획했던 모든 기능을 구현하지 못함
      • mongoDB 미적용: mongoDB를 배운 후 발표까지의 시간이 짧아 데이터 관리에 mongoDB를 직접 사용해보지 못함
      • 발표 목적 파악 미흡: 발표의 목적을 잘 파악하지 못해 개발 작업에 대한 설명이 부족하였음
    • 잘한 부분
      • 원활한 의사소통: 팀원 간의 의사소통이 원활히 잘 이루어져 피드백과 수정이 금방 이루어짐
      • 적절한 역할 분배: 역할 분배가 잘 이루어져 각자의 작업에 더욱 집중할 수 있었음
      • 낯선 기술 습득: 어렵고 막막했던 기술들을 팀원들 모두 도전해 보고 결과물을 만들어냄
      • 적극적인 참여: 팀원들 모두가 프로젝트에 적극적으로 참여하고 작업을 구상해냄
    • 배운 점 및 주관적인 의견
      • 원하던 모든 기능을 구현하지는 못했지만, 기능에 관한 자료들을 찾아보고 시도해 보는 경험도 값졌다고 생각한다. 아직 공부가 많이 필요하다는 걸 깨달았고, 의지를 더 다질 수 있었다.
      • 기획부터 디자인, 구현까지 모두 해보는 경험 자체가 즐거웠고 만족할만한 결과물을 만들어내어 자신감을 얻게 되었다.
      • 달력 기능을 구현하며 다양한 라이브러리 및 플러그인을 찾아보게 되었고, 기능에 따른 캘린더 관련 라이브러리에 대해 많이 알게 되었다.
      • localStorage에 대한 개념을 제대로 알게 되었고 곧바로 활용까지 하여 잘 이해할 수 있었다.
      • 만족스러운 결과물을 얻게 된 데에는 원활한 의사소통의 역할이 가장 컸다고 생각한다. 이번 프로젝트를 계기로 또 다시 의사소통의 중요성을 깨닫게 되었다.
      • 처음 계획할 때 주어진 시간과 각자의 능력을 고려하여 계획을 했으면 더 효율적으로 시간을 사용할 수 있었을 것 같다는 아쉬움이 남는다.
