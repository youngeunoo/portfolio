    lilybyred - burn & heat

    코스메틱 브랜드인 “릴리바이레드”의 “burn & heat” 컬렉션과 그 상품들의 프로모션 페이지를 디자인, 구상하고 다양한 효과와 함께 구현한다.

    개인 프로젝트 개요

    • 작업기간 : 24. 01. 04 - 24. 01. 16
    • 기여도 : 100%

    목표

    • 프로모션 페이지를 리디자인하며 디자인에 대한 이해를 높임
    • GSAP ScrollTrigger를 활용하며 GSAP에 대한 개념을 익히고 적절하게 사용할 수 있도록 함
    • 이미지 요소를 디자인과 레이아웃에 어울리게 사용할 수 있는 능력을 기름

    작업 순서

    • 사전 기획
      • 리브랜딩 브랜드/상품 선정
    • 디자인 작업
      • 필요한 이미지, 아이콘 수집
      • 메인 색상과 포인트 색상, 사용될 폰트 선정
      • 디자인 키워드 선정
      • 디자인 완성
    • 개발 작업
      • 페이지 레이아웃 작업
      • 각 페이지에 필요한 기능 구현

    사용 기술 / 도구

    • HTML5
    • CSS3
    • JaveScript
    • GSAP
    • Figma

    디자인 가이드

    • 키워드
      • #뜨거움
      • #열정적
      • #강렬함
    • 색상

      • 메인 색상

      • 폰트 색상

      • → 강렬함과 열정적인 느낌을 주면서도 따뜻한 느낌을 주기 위한 색상 선정
    • 폰트
    • 이미지 요소
      • → 붉은 색상, 불꽃과 온도를 상징하는 아이콘을 사용하여 분위기를 시각화

    퍼블리싱

    • GSAP의 callback 함수를 사용하여 스크롤이 다양하게 영역을 넘나들어도 자연스럽게 효과가 적용되고 변화할 수 있게 하였다. 
      
            onEnter: () => {
              gsap.to("body", {
                backgroundColor: "#d61010",
              });
              gsap.to(".warning p", {
                animation: "blink 1s infinite",
              });
              gsap.to(".warning-wrap", {
                opacity: 1,
              });
            },
            onEnterBack: () => {
              gsap.to("body", {
                backgroundColor: "#d61010",
              });
              gsap.to(".warning-wrap", {
                opacity: 1,
              });
            },
            onLeave: () => {
              gsap.to(".warning-wrap", {
                opacity: 0,
              });
            },
            onLeaveBack: () => {
              gsap.to(".warning-wrap", {
                opacity: 0,
              });
            },
    • "Lenis" 플러그인을 사용하여 부드러운 스크롤을 만들어 사용자가 페이지를 보기에 더 편하도록 했다.
    • ::selection과 ::-webkit-scrollbar 등을 사용하여 페이지의 개성을 더했다.

    문제 및 해결

    • 영역과 영역이 넘어갈 때 경계가 눈에 보여 자연스럽지 못했다.

      → ScrollTrigger를 사용하여 영역이 넘어갈 때 body의 backgroundColor에 변화를 주어 자연스럽게 변화하도록 했다.

      
gsap.timeline({ScrollTrigger:{
  trigger:".warning",
  .
  .
  onEnter: () => {
    gsap.to("body", {
      backgroundColor:"#d61010",
      });
    },
  },
})
    • process 영역의 이미지들의 구조가 밋밋하고, 아무런 특징이 드러나지 않았다.

      → 브라우저와 폴더 느낌의 프레임 안에 이미지를 배치한 후, 그에 어울리는 ScrollTrigger 효과를 주어 해결했다.

    • 스크롤이 부드럽고 천천히 내려가게 하고 싶어 다양한 시도를 해보았지만 원하던 효과가 나지 않았다.

      → Lenis라는 플러그인을 발견하여 적용했더니 원하던 스크롤 느낌이 났다.

    후기

    • 아쉬운 부분
      • 효과의 다양성 부족: 더욱 다양한 효과를 적용하여 사용자의 흥미를 이끌어내고 싶었지만 그렇지 못함
      • 기술 활용 미숙: ScrollTrigger 활용에 아직은 미숙하여 연결이 100% 자연스럽지 못함
      • 일러스트 활용 실패: 다양한 일러스트를 사용하여 작업하고 싶었지만 능력 부족으로 작업하지 못함
    • 잘한 부분
      • 원하던 효과 구현: 처음 디자인을 하며 구상했던 움직임 효과를 모두 성공적으로 구현해냄
      • 디자인적 능력 향상: 직접 처음부터 끝까지 디자인과 레이아웃을 구성해보며, 그에 대한 능력을 기를 수 있었음
      • 기술 구현: 스크롤 제어 등 구현이 힘들었던 기능들도 끝까지 찾고 실행해보며 성공해냄
    • 배운 점 및 주관적인 의견
      • 온전히 내가 기획하고 구성한 페이지 하나를 만들어 내는 것 자체가 즐거웠다. 특히 모든 움직임과 기능을 혼자 완성해냈다는 점에서 뿌듯함이 많이 느껴진다.
      • 레이아웃 구성이 생각보다 쉽지 않다는 걸 느꼈다. 어떤 요소를 어디에 어떻게 배치해야 할지 굉장히 고민했고, 막상 배치해보면 생각하던 느낌과 다를 때가 많아 조금 힘들었다.
      • 페이지 곳곳에 사용된 아이템들을 기존에 있던 이미지들로만 구성하려니 한계가 느껴지는 것 같았다. 풍성한 페이지 구성을 위해서는 다양한 툴을 다루고 사용하는 법을 익히는 게 좋을 것 같다는 생각을 많이 하게 되는 계기였다.
